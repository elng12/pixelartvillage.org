# 伪本地化验证步骤

## 🚀 开发服务器已启动

开发服务器现在应该正在运行，并且已启用伪本地化模式。

## 📝 验证步骤

### 1. 打开浏览器
访问开发服务器的地址（通常是以下之一）：
- http://localhost:5173
- http://localhost:3000
- http://127.0.0.1:5173

### 2. 切换到伪本地化语言
1. 在页面右上角找到**语言选择器**（Language selector）
2. 点击下拉菜单
3. 选择 **"pseudo"** 语言

### 3. 检查页面文本

浏览整个页面，检查以下区域：

#### ✅ 应该显示伪本地化格式（正确）
所有文本应该显示为：
```
［‼ Ťēẋŧ Ŵįŧħ Śƥēćįåŀ Čħåŗåćŧēŗś ‼］
```

#### ❌ 硬编码文本（需要修复）
如果看到正常的英文文本，说明有硬编码：
```
Normal English Text
```

### 4. 需要检查的关键区域

- [ ] **导航栏** - 所有链接文本
- [ ] **主页标题** - "Image to Pixel Art Converter"
- [ ] **Features Section** - 三个功能卡片（Palette control, Live adjustments, 100% client-side）
- [ ] **How It Works** - 三个步骤说明
- [ ] **Showcase Section** - 描述文本
- [ ] **FAQ Section** - 所有问题和答案
- [ ] **Related Links** - 转换器和指南列表
- [ ] **Footer** - 所有链接和文本
- [ ] **Contact 页面** - 所有段落和链接
- [ ] **Editor 中的 PaletteManager** - 所有按钮和标签

### 5. 如果发现硬编码文本

如果发现任何未标记的英文文本，请记录：
- 所在的页面/组件
- 具体的文本内容
- 在页面上的位置

然后运行：
```powershell
# 修复代码后，同步翻译键
node scripts/sync-translation-keys.cjs

# 重新生成伪本地化
npm run i18n:pseudo

# 验证翻译键一致性
npm run i18n:check
```

## 🎯 验证完成标准

所有面向用户的文本都应该：
1. ✅ 通过 `useTranslation()` Hook 获取
2. ✅ 使用 `t('translation.key')` 渲染
3. ✅ 在伪本地化模式下显示为 `［‼ ... ‼］` 格式
4. ✅ 在所有支持的语言中都有对应的翻译键
