# 首页 SEO 速读版（2026-03-25）

## 一句话结论

首页不是“没配 SEO”，而是“基础标签都在，但配置来源不统一，页面信号又偏杂”，所以更像是质量和聚焦问题，不是单纯缺标签问题。

## 现在情况

- 线上首页已经有标题、简介、规范地址，也就是 `canonical`、多语言对应，也就是 `hreflang`、以及社交分享卡片。
- 当前线上真正生效的是预渲染后的页面，不是仓库根目录那份老 `index.html`。
- 首页标题方向是对的，已经比旧版本更聚焦主工具意图。

## 主要问题

- 团队很容易看错文件：代码里看到的一套，不一定是 Google 实际看到的一套。
- 首页文案重复关键词偏多，读起来更像在“堆词”，不像自然介绍。
- 首页承载了太多泛意图，同时还挂了很多第三方徽章外链，信号比较嘈杂。
- React 里的 SEO 组件接口和真实行为不一致，看起来支持很多 SEO 字段，实际上运行时只会改标题和语言。

## 影响

- 这类问题不会让页面“直接消失”，但会让 Google 更难判断首页到底是不是最值得排名的主工具页。
- 结果通常不是“完全不收录”，而是泛词排名不稳、展示下降、点击跟着掉。

## 现在最该做的 3 件事

1. 统一首页 SEO 的唯一权威来源，别再同时参考两套配置。
2. 清理首页上的大部分第三方徽章外链，先把页面信号降噪。
3. 把首页文案改得更自然、更聚焦，只服务一个主意图：主转换器入口。

## 现在不该做的事

- 不要继续往首页加更多关键词。
- 不要一边改标题，一边改结构，一边改内链，最后不知道哪一步起作用。
- 不要先怀疑“是不是少了 title 或 description”，因为这些基础项现在已经有了。

## 建议节奏

- 第一阶段：先统一配置来源，清理外链噪音。
- 第二阶段：再改首页 H1 和 hero 文案，减少重复关键词。
- 第三阶段：部署后看 GSC，只盯首页页面和核心查询变化。

## 给老板的话

这不是一个“补几个 SEO 标签就能解决”的问题。基础设置已经有了，下一步重点是让首页更像一个清晰、可信、聚焦的主工具页。
